## 2. Table: `training_programs`

Stores the training program definitions — reusable templates for training sessions (e.g., "HSE Induction", "Fire Safety", "Forklift Certification").

### Column Definitions

| # | Column | Type | Nullable | Default | Constraints / Notes |
|---|--------|------|----------|---------|---------------------|
| 1 | `id` | `bigint` | NO | — | **Primary Key**, auto-increment |
| 2 | `code` | `varchar(50)` | NO | — | **Unique**. Human-readable program code (e.g., `HSE-IND`, `FIRE-01`) |
| 3 | `name` | `varchar(255)` | NO | — | Program name (Indonesian label) |
| 4 | `description` | `text` | YES | `NULL` | Detailed description of the training |
| 5 | `category` | `varchar(50)` | NO | — | Category: `safety`, `technical`, `compliance`, `soft_skill`, `environment`, `security`, `quality`, `first_aid` |
| 6 | `duration_hours` | `integer` | NO | — | Duration in hours (e.g., 8, 16, 24) |
| 7 | `is_certification` | `boolean` | NO | `false` | Whether this program issues a certificate upon completion |
| 8 | `validity_months` | `integer` | YES | `NULL` | Certificate validity in months. NULL = no expiry. Used to auto-calculate `expiry_date` on records. |
| 9 | `is_active` | `boolean` | NO | `true` | Active flag. Inactive programs cannot be selected for new records. |
| 10 | `created_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed |
| 11 | `updated_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed, auto-updated on save |

### PostgreSQL DDL (Reference)

```sql
CREATE TABLE training_programs (
    id                BIGSERIAL       PRIMARY KEY,
    code              VARCHAR(50)     NOT NULL UNIQUE,
    name              VARCHAR(255)    NOT NULL,
    description       TEXT            NULL,
    category          VARCHAR(50)     NOT NULL,
    duration_hours    INTEGER         NOT NULL,
    is_certification  BOOLEAN         NOT NULL DEFAULT false,
    validity_months   INTEGER         NULL,
    is_active         BOOLEAN         NOT NULL DEFAULT true,
    created_at        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT training_programs_category_check CHECK (
        category IN (
            'safety',
            'technical',
            'compliance',
            'soft_skill',
            'environment',
            'security',
            'quality',
            'first_aid'
        )
    ),

    CONSTRAINT training_programs_validity_check CHECK (
        validity_months IS NULL OR validity_months > 0
    ),

    CONSTRAINT training_programs_duration_check CHECK (
        duration_hours > 0
    )
);
```

### Laravel Migration (Reference)

```php
Schema::create('training_programs', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->string('code', 50)->unique();
    $table->string('name', 255);
    $table->text('description')->nullable();
    $table->string('category', 50);
    $table->integer('duration_hours');
    $table->boolean('is_certification')->default(false);
    $table->integer('validity_months')->nullable();
    $table->boolean('is_active')->default(true);
    $table->timestamps();

    $table->check("category IN ('safety','technical','compliance','soft_skill','environment','security','quality','first_aid')", 'training_programs_category_check');
});
```

### Design Notes

- **No soft deletes** (`deleted_at`) — programs are deactivated via `is_active = false` instead of deleted.
- **`code`** is unique and human-readable. Used in program selection dropdowns.
- **`validity_months`** is NULL for programs that do not issue expiring certificates (e.g., one-time inductions).
- **`is_certification`** flag determines whether a certificate file is expected when the record is completed.
- **`category`** is stored as `varchar` with a CHECK constraint — simplifies application-level validation.

## 3. Table: `training_records`

Stores individual training records — one employee attending one program. Each record has a unique `training_number` generated by `NumberingService`.

### Column Definitions

| # | Column | Type | Nullable | Default | Constraints / Notes |
|---|--------|------|----------|---------|---------------------|
| 1 | `id` | `bigint` | NO | — | **Primary Key**, auto-increment |
| 2 | `training_number` | `varchar(50)` | NO | — | **Unique**. Auto-generated at create (format: `TRN-{YYYY}-{0001}`) |
| 3 | `employee_id` | `bigint` | NO | — | **FK → `employees.id`**. Employee who received the training |
| 4 | `training_program_id` | `bigint` | NO | — | **FK → `training_programs.id`**. The program this record belongs to |
| 5 | `provider` | `varchar(255)` | YES | `NULL` | Training provider/institution (e.g., external vendor) |
| 6 | `start_date` | `date` | NO | — | Training start date |
| 7 | `end_date` | `date` | YES | `NULL` | Training end date (nullable for ongoing/single-day) |
| 8 | `status` | `varchar(50)` | NO | `'scheduled'` | Lifecycle state: `scheduled`, `in_progress`, `completed`, `expired`, `cancelled` |
| 9 | `score` | `decimal(5,2)` | YES | `NULL` | Exam/assessment score (e.g., 85.50) |
| 10 | `result` | `varchar(20)` | YES | `NULL` | Result: `pass`, `fail`, `pending` |
| 11 | `certificate_number` | `varchar(255)` | YES | `NULL` | External certificate number from provider |
| 12 | `certificate_file_id` | `bigint` | YES | `NULL` | **FK → `managed_files.id`** (nullable). Links to uploaded certificate file. |
| 13 | `expiry_date` | `date` | YES | `NULL` | Certificate expiry date. Auto-calculated from `end_date + validity_months` if program has validity. |
| 14 | `notes` | `text` | YES | `NULL` | Free-text notes |
| 15 | `created_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed |
| 16 | `updated_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed, auto-updated on save |

### PostgreSQL DDL (Reference)

```sql
CREATE TABLE training_records (
    id                      BIGSERIAL       PRIMARY KEY,
    training_number         VARCHAR(50)     NOT NULL UNIQUE,
    employee_id             BIGINT          NOT NULL REFERENCES employees(id),
    training_program_id     BIGINT          NOT NULL REFERENCES training_programs(id),
    provider                VARCHAR(255)    NULL,
    start_date              DATE            NOT NULL,
    end_date                DATE            NULL,
    status                  VARCHAR(50)     NOT NULL DEFAULT 'scheduled',
    score                   DECIMAL(5,2)    NULL,
    result                  VARCHAR(20)     NULL,
    certificate_number      VARCHAR(255)    NULL,
    certificate_file_id     BIGINT          NULL REFERENCES managed_files(id),
    expiry_date             DATE            NULL,
    notes                   TEXT            NULL,
    created_at              TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at              TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT training_records_status_check CHECK (
        status IN (
            'scheduled',
            'in_progress',
            'completed',
            'expired',
            'cancelled'
        )
    ),

    CONSTRAINT training_records_result_check CHECK (
        result IS NULL OR result IN ('pass', 'fail', 'pending')
    ),

    CONSTRAINT training_records_score_check CHECK (
        score IS NULL OR (score >= 0 AND score <= 100)
    ),

    CONSTRAINT training_records_dates_check CHECK (
        end_date IS NULL OR end_date >= start_date
    )
);
```

### Laravel Migration (Reference)

```php
Schema::create('training_records', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->string('training_number', 50)->unique();
    $table->foreignId('employee_id')->constrained('employees');
    $table->foreignId('training_program_id')->constrained('training_programs');
    $table->string('provider', 255)->nullable();
    $table->date('start_date');
    $table->date('end_date')->nullable();
    $table->string('status', 50)->default('scheduled');
    $table->decimal('score', 5, 2)->nullable();
    $table->string('result', 20)->nullable();
    $table->string('certificate_number', 255)->nullable();
    $table->foreignId('certificate_file_id')->nullable()->constrained('managed_files');
    $table->date('expiry_date')->nullable();
    $table->text('notes')->nullable();
    $table->timestamps();

    $table->check("status IN ('scheduled','in_progress','completed','expired','cancelled')", 'training_records_status_check');
    $table->check("result IS NULL OR result IN ('pass','fail','pending')", 'training_records_result_check');
});
```

### Design Notes

- **No soft deletes** (`deleted_at`) — training records are never deleted. Use `status = 'cancelled'` instead.
- **`training_number`** is generated at **create** (not on submit), so every record has a number immediately.
- **`certificate_file_id`** is a direct FK to `managed_files.id`. This is in addition to the polymorphic `module_name='training'` + `reference_id` pattern used by the file service for querying. The direct FK allows efficient joins for loading certificate details.
- **`expiry_date`** is auto-calculated when `end_date` is set and the program has `validity_months`. Can also be set manually.
- **`status`** is a simple string field with manual transitions — no `workflow_instances` or `workflow_histories` for this module.
- **`score`** is `decimal(5,2)` — allows values 0.00 to 100.00 (or 0–999.99 if custom scale needed). CHECK constraint enforces 0–100 range.

## 6. Shared Relations

The Training & Competency module uses the **polymorphic `module_name + reference_id`** pattern for cross-cutting concerns. For this module:

- `module_name = 'training'`
- `reference_id = training_records.id`

### 6.1 Managed Files (`managed_files`)

Certificate files for training records.

| Column | Value |
|---|---|
| `module_name` | `'training'` |
| `reference_id` | `training_records.id` |
| `collection` | `'certificate'` |
| `uploaded_by` | `users.id` (FK) |

```
training_records.id ──► managed_files.reference_id
                         managed_files.module_name = 'training'
                         managed_files.collection = 'certificate'
```

**Usage**: `TrainingRecord::certificate()` returns the file where `module_name='training'` AND `reference_id=$this->id` AND `collection='certificate'`.

Additionally, `training_records.certificate_file_id` provides a direct FK for efficient eager loading.

### 6.2 Activity Logs (`activity_logs`)

Timeline of actions performed on a training record (created, updated, status changed, etc.).

| Column | Value |
|---|---|
| `module_name` | `'training'` |
| `reference_id` | `training_records.id` |
| `event` | `'created'`, `'updated'`, `'status_changed'`, `'file_uploaded'`, etc. |
| `actor_id` | `users.id` (FK) |
| `properties` | JSON (before/after snapshot, metadata) |

```
training_records.id ──► activity_logs.reference_id
                         activity_logs.module_name = 'training'
```

**Usage**: `TrainingRecord::activities()` returns all activity log entries for this record.

### 6.3 Audit Logs (`audit_logs`)

Detailed audit trail of field-level changes on training records.

| Column | Value |
|---|---|
| `module_name` | `'training'` |
| `reference_id` | `training_records.id` |
| `auditable_type` | `'TrainingRecord'` (or fully-qualified model class) |
| `auditable_id` | `training_records.id` |
| `old_values` | JSON (previous field values) |
| `new_values` | JSON (new field values) |
| `actor_id` | `users.id` (FK) |

### 6.4 Notifications (`core_notifications`)

Expiry reminders and training assignment notifications.

| Column | Value |
|---|---|
| `module_name` | `'training'` |
| `reference_id` | `training_records.id` |
| `type` | `'training.expiry_reminder_30d'`, `'training.expiry_reminder_7d'`, `'training.record_created'` |

### Shared Relations Summary

```
                          ┌──────────────────┐
                          │ training_records  │
                          │ (id: PK)          │
                          └────────┬──────────┘
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
              module_name='training'              │
              reference_id=training_records.id     │
                    │              │              │
    ┌───────────────▼──┐  ┌───────▼──────┐  ┌───▼──────────────┐
    │  managed_files    │  │ activity_logs │  │  audit_logs     │
    │  (certificate)    │  │  (timeline)   │  │  (field changes) │
    └──────────────────┘  └──────────────┘  └─────────────────┘
                    │
    ┌───────────────▼──┐
    │ core_notifications│
    │ (expiry reminders)│
    └──────────────────┘

    All linked via: module_name='training' AND reference_id=training_records.id
    No hard FKs — application-layer validated polymorphic relation.
    (certificate_file_id is a direct FK exception for efficient loading)
```

## 7. Migration File Naming Convention

### Convention

Migrations follow Laravel's standard naming pattern:

```
{YYYY}_{MM}_{DD}_{HHMMSS}_{verb}_{table_name}_table.php
```

### Migration Files for This Module

|---|---|---|

| 2 | `YYYY_MM_DD_HHMMSS_create_training_records_table.php` | Creates `training_records` table |

### Model Relationships

```php
// TrainingProgram.php
public function records(): HasMany
{
    return $this->hasMany(TrainingRecord::class, 'training_program_id');
}

// TrainingRecord.php
public function employee(): BelongsTo
{
    return $this->belongsTo(Employee::class);
}

public function program(): BelongsTo
{
    return $this->belongsTo(TrainingProgram::class, 'training_program_id');
}

public function certificateFile(): BelongsTo
{
    return $this->belongsTo(ManagedFile::class, 'certificate_file_id');
}

// Polymorphic: activity logs
public function activities(): HasMany
{
    return $this->hasMany(ActivityLog::class, 'reference_id')
        ->where('module_name', 'training');
}

// Polymorphic: audit logs
public function audits(): HasMany
{
    return $this->hasMany(AuditLog::class, 'reference_id')
        ->where('module_name', 'training');
}
```
