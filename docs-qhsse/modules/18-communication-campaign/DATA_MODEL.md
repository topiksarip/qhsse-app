## 2. Table: `campaigns`

Stores the communication campaigns — safety alerts, lesson learned, campaigns, announcements, and newsletters. Each campaign has a unique `campaign_number` generated by `NumberingService`.

### Column Definitions

| # | Column | Type | Nullable | Default | Constraints / Notes |
|---|--------|------|----------|---------|---------------------|
| 1 | `id` | `bigint` | NO | — | **Primary Key**, auto-increment |
| 2 | `campaign_number` | `varchar(50)` | NO | — | **Unique**. Auto-generated at create (format: `COM-{YYYY}-{0001}`) |
| 3 | `title` | `varchar(255)` | NO | — | Campaign title (Indonesian label) |
| 4 | `type` | `varchar(50)` | NO | — | Type: `safety_alert`, `lesson_learned`, `campaign`, `announcement`, `newsletter` |
| 5 | `content` | `text` | NO | — | Rich text content of the campaign (HTML from rich text editor) |
| 6 | `target_audience` | `varchar(50)` | NO | `'all'` | Target audience: `all`, `specific_site`, `specific_department`, `specific_role` |
| 7 | `site_id` | `bigint` | YES | `NULL` | **FK → `sites.id`**. Required when `target_audience = 'specific_site'`. NULL otherwise. |
| 8 | `department_id` | `bigint` | YES | `NULL` | **FK → `departments.id`**. Required when `target_audience = 'specific_department'`. NULL otherwise. |
| 9 | `target_role` | `varchar(100)` | YES | `NULL` | Role name. Required when `target_audience = 'specific_role'`. NULL otherwise. |
| 10 | `published_at` | `timestamp` | YES | `NULL` | Timestamp when campaign was published. NULL while in draft. |
| 11 | `expires_at` | `timestamp` | YES | `NULL` | Expiry timestamp. NULL = no expiry. |
| 12 | `status` | `varchar(50)` | NO | `'draft'` | Lifecycle state: `draft`, `published` |
| 13 | `author_id` | `bigint` | NO | — | **FK → `users.id`**. User who created the campaign |
| 14 | `view_count` | `integer` | NO | `0` | Total unique views (incremented on detail page load with deduplication) |
| 15 | `created_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed |
| 16 | `updated_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed, auto-updated on save |

### PostgreSQL DDL (Reference)

```sql
CREATE TABLE campaigns (
    id                  BIGSERIAL       PRIMARY KEY,
    campaign_number     VARCHAR(50)     NOT NULL UNIQUE,
    title               VARCHAR(255)    NOT NULL,
    type                VARCHAR(50)     NOT NULL,
    content             TEXT            NOT NULL,
    target_audience     VARCHAR(50)     NOT NULL DEFAULT 'all',
    site_id             BIGINT          NULL REFERENCES sites(id),
    department_id       BIGINT          NULL REFERENCES departments(id),
    target_role         VARCHAR(100)    NULL,
    published_at        TIMESTAMP       NULL,
    expires_at          TIMESTAMP       NULL,
    status              VARCHAR(50)     NOT NULL DEFAULT 'draft',
    author_id           BIGINT          NOT NULL REFERENCES users(id),
    view_count          INTEGER         NOT NULL DEFAULT 0,
    created_at          TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT campaigns_type_check CHECK (
        type IN (
            'safety_alert',
            'lesson_learned',
            'campaign',
            'announcement',
            'newsletter'
        )
    ),

    CONSTRAINT campaigns_target_audience_check CHECK (
        target_audience IN (
            'all',
            'specific_site',
            'specific_department',
            'specific_role'
        )
    ),

    CONSTRAINT campaigns_status_check CHECK (
        status IN ('draft', 'published')
    ),

    CONSTRAINT campaigns_view_count_check CHECK (
        view_count >= 0
    ),

    -- Conditional FK enforcement: site_id required when target_audience = 'specific_site'
    CONSTRAINT campaigns_site_required_check CHECK (
        (target_audience <> 'specific_site') OR (site_id IS NOT NULL)
    ),

    -- department_id required when target_audience = 'specific_department'
    CONSTRAINT campaigns_dept_required_check CHECK (
        (target_audience <> 'specific_department') OR (department_id IS NOT NULL)
    ),

    -- target_role required when target_audience = 'specific_role'
    CONSTRAINT campaigns_role_required_check CHECK (
        (target_audience <> 'specific_role') OR (target_role IS NOT NULL)
    ),

    -- site_id must be NULL when target_audience <> 'specific_site'
    CONSTRAINT campaigns_site_null_check CHECK (
        (target_audience = 'specific_site') OR (site_id IS NULL)
    ),

    -- department_id must be NULL when target_audience <> 'specific_department'
    CONSTRAINT campaigns_dept_null_check CHECK (
        (target_audience = 'specific_department') OR (department_id IS NULL)
    ),

    -- target_role must be NULL when target_audience <> 'specific_role'
    CONSTRAINT campaigns_role_null_check CHECK (
        (target_audience = 'specific_role') OR (target_role IS NULL)
    )
);
```

### Laravel Migration (Reference)

```php
Schema::create('campaigns', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->string('campaign_number', 50)->unique();
    $table->string('title', 255);
    $table->string('type', 50);
    $table->text('content');
    $table->string('target_audience', 50)->default('all');
    $table->foreignId('site_id')->nullable()->constrained('sites');
    $table->foreignId('department_id')->nullable()->constrained('departments');
    $table->string('target_role', 100)->nullable();
    $table->timestamp('published_at')->nullable();
    $table->timestamp('expires_at')->nullable();
    $table->string('status', 50)->default('draft');
    $table->foreignId('author_id')->constrained('users');
    $table->integer('view_count')->default(0);
    $table->timestamps();

    $table->check("type IN ('safety_alert','lesson_learned','campaign','announcement','newsletter')", 'campaigns_type_check');
    $table->check("target_audience IN ('all','specific_site','specific_department','specific_role')", 'campaigns_target_audience_check');
    $table->check("status IN ('draft','published')", 'campaigns_status_check');
    $table->check('view_count >= 0', 'campaigns_view_count_check');
});
```

### Design Notes

- **No soft deletes** (`deleted_at`) — campaigns are never deleted. Once published, they remain as historical records.
- **`campaign_number`** is generated at **create** (not on publish), so every campaign has a number immediately.
- **`content`** stores rich text HTML from the editor (e.g., TipTap output). Sanitized on render.
- **`target_audience`** + conditional fields (`site_id`, `department_id`, `target_role`) enforce mutual exclusivity at the database level via CHECK constraints.
- **`status`** is a simple string field with manual transitions — `draft` → `published`. No `workflow_instances` or `workflow_histories` for this module.
- **`view_count`** is a denormalized counter incremented on detail page load with per-user deduplication (via cache/session).
- **`published_at`** is set only when the publish action is triggered. NULL while in draft.
- **`expires_at`** is optional. When set and past, the campaign is marked "kedaluwarsa" in UI but remains accessible.

## 3. Table: `campaign_acknowledgments`

Stores individual acknowledgment records — one user confirming they have read a campaign. Each acknowledgment is unique per campaign+user combination.

### Column Definitions

| # | Column | Type | Nullable | Default | Constraints / Notes |
|---|--------|------|----------|---------|---------------------|
| 1 | `id` | `bigint` | NO | — | **Primary Key**, auto-increment |
| 2 | `campaign_id` | `bigint` | NO | — | **FK → `campaigns.id`** (ON DELETE CASCADE). Campaign being acknowledged |
| 3 | `user_id` | `bigint` | NO | — | **FK → `users.id`**. User who acknowledged |
| 4 | `acknowledged_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Timestamp of acknowledgment |
| 5 | `ip_address` | `varchar(45)` | YES | `NULL` | IP address of the user when acknowledging (IPv4 or IPv6, for audit trail) |
| 6 | `created_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed |
| 7 | `updated_at` | `timestamp` | NO | `CURRENT_TIMESTAMP` | Laravel managed, auto-updated on save |

### PostgreSQL DDL (Reference)

```sql
CREATE TABLE campaign_acknowledgments (
    id                BIGSERIAL       PRIMARY KEY,
    campaign_id       BIGINT          NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
    user_id           BIGINT          NOT NULL REFERENCES users(id),
    acknowledged_at   TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    ip_address        VARCHAR(45)     NULL,
    created_at        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT campaign_acknowledgments_campaign_user_unique UNIQUE (campaign_id, user_id)
);
```

### Laravel Migration (Reference)

```php
Schema::create('campaign_acknowledgments', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->foreignId('campaign_id')->constrained('campaigns')->cascadeOnDelete();
    $table->foreignId('user_id')->constrained('users');
    $table->timestamp('acknowledged_at');
    $table->string('ip_address', 45)->nullable();
    $table->timestamps();

    // Prevent double acknowledgment — one user per campaign
    $table->unique(['campaign_id', 'user_id'], 'campaign_acknowledgments_campaign_user_unique');
});
```

### Design Notes

- **Unique constraint** on `(campaign_id, user_id)` — a user can only acknowledge a campaign once.
- **ON DELETE CASCADE** on `campaign_id` — if a campaign is deleted (hypothetically), all its acknowledgments are automatically removed.
- **`acknowledged_at`** is separate from `created_at` for semantic clarity — it represents the moment the user clicked "Acknowledge".
- **`ip_address`** is stored for audit trail purposes (compliance). Supports both IPv4 (max 15 chars) and IPv6 (max 45 chars).
- **No soft deletes** — acknowledgments are permanent records once created.

## 6. Shared Relations

The Communication & Campaign module uses the **polymorphic `module_name + reference_id`** pattern for cross-cutting concerns. For this module:

- `module_name = 'communication'`
- `reference_id = campaigns.id`

### 6.1 Managed Files (`managed_files`)

Attachment files for campaigns.

| Column | Value |
|---|---|
| `module_name` | `'communication'` |
| `reference_id` | `campaigns.id` |
| `collection` | `'attachment'` |
| `uploaded_by` | `users.id` (FK) |

```
campaigns.id ──► managed_files.reference_id
                 managed_files.module_name = 'communication'
                 managed_files.collection = 'attachment'
```

**Usage**: `Campaign::attachments()` returns files where `module_name='communication'` AND `reference_id=$this->id` AND `collection='attachment'`.

### 6.2 Activity Logs (`activity_logs`)

Timeline of actions performed on a campaign (created, updated, published, acknowledged, etc.).

| Column | Value |
|---|---|
| `module_name` | `'communication'` |
| `reference_id` | `campaigns.id` |
| `event` | `'created'`, `'updated'`, `'published'`, `'acknowledged'`, `'file_uploaded'`, etc. |
| `actor_id` | `users.id` (FK) |
| `properties` | JSON (before/after snapshot, metadata) |

```
campaigns.id ──► activity_logs.reference_id
                 activity_logs.module_name = 'communication'
```

**Usage**: `Campaign::activities()` returns all activity log entries for this campaign.

### 6.3 Audit Logs (`audit_logs`)

Detailed audit trail of field-level changes on campaigns.

| Column | Value |
|---|---|
| `module_name` | `'communication'` |
| `reference_id` | `campaigns.id` |
| `auditable_type` | `'Campaign'` (or fully-qualified model class) |
| `auditable_id` | `campaigns.id` |
| `old_values` | JSON (previous field values) |
| `new_values` | JSON (new field values) |
| `actor_id` | `users.id` (FK) |

### 6.4 Notifications (`core_notifications`)

Campaign published notifications and acknowledgment reminders.

| Column | Value |
|---|---|
| `module_name` | `'communication'` |
| `reference_id` | `campaigns.id` |
| `type` | `'communication.campaign_published'`, `'communication.acknowledgment_reminder'` |

// Helper: check if user has acknowledged
public function isAcknowledgedBy(User $user): bool
{
    return $this->acknowledgments()
        ->where('user_id', $user->id)
        ->exists();
}

// Helper: check if user is in target audience
public function isTargetedAt(User $user): bool
{
    return match ($this->target_audience) {
        'all' => true,
        'specific_site' => $user->employee?->site_id === $this->site_id,
        'specific_department' => $user->employee?->department_id === $this->department_id,
        'specific_role' => $user->roles()->where('name', $this->target_role)->exists(),
    };
}
