# Project Charter — QHSSE Web Application

## 1. Maksud & Tujuan
Aplikasi web modular untuk mengelola Quality, Health, Safety, Security, dan Environment (QHSSE)
secara terstruktur, terdokumentasi, dan dapat diaudit. Menggantikan pencatatan manual/spreadsheet
dengan sistem kerja QHSSE sungguhan: insiden, investigasi, CAPA, inspeksi, audit, dokumen,
training, permit, pemantauan lingkungan, dan keamanan.

## 2. Ruang Lingkup
- **In scope**: 19 modul bisnis + platform core (RBAC, master data, file pribadi, numbering, workflow, audit trail, komentar, notifikasi, dashboard, ekspor).
- **Out of scope (early phases)**: microservices, plugin system kompleks, multi-tenant eksternal.

## 3. Prinsip Produk
Reliable · Auditable · Practical · Controlled · Traceable · Modular · Production-minded.

## 4. Stakeholder
- QHSSE Manager / Officer — pengelola program.
- Supervisor / Foreman / Operator — pelapor & eksekutor lapangan.
- Department Head / Top Management — approval & visibility.
- Security Officer, Auditor, Contractor — peran terbatas.

## 5. Metrik Sukses
- Waktu pelaporan insiden turun; data terlacak; audit trail lengkap; dashboard KPI dapat dipercaya.
