## 2. Context

Setelah deployment P0 dan audit komprehensif terhadap UI/menu/RBAC, ditemukan tiga gap kecil yang dapat diperbaiki dengan cepat:

1. **Submenu Training/Emergency tidak tersedia**: Program Pelatihan, Record Pelatihan, Matriks Kompetensi, Rencana Darurat, Latihan Darurat, dan Kontak Darurat memiliki UI/route/controller tetapi tidak ada menu top-level, sehingga sulit ditemukan user.
2. **Training permission tidak diberikan ke QHSSE roles**: QHSSE Manager, QHSSE Officer, dan Supervisor tidak memiliki permission Training sama sekali, padahal secara operasional mereka perlu akses.
3. **Emergency routes tidak dibungkus auth middleware**: Emergency route hanya mengandalkan controller policy tanpa route middleware, tidak konsisten dengan modul lain dan tidak mengikuti defense-in-depth.

P0.1 menutup ketiga gap tersebut dengan empat track: menu submenu, role mapping Training, Emergency route auth, dan regression test.

## 3. Scope Dikerjakan

### Track 1: Tambahkan 6 Submenu

- Memecah menu `Training & Competency` menjadi tiga item:
  - Program Pelatihan (`training.programs.index`, permission `training.programs.view`)
  - Record Pelatihan (`training.records.index`, permission `training.records.view`)
  - Matriks Kompetensi (`training.matrix.index`, permission `training.records.view`)
- Memecah menu `Emergency Preparedness` menjadi tiga item:
  - Rencana Darurat (`emergency.plans.index`, permission `emergency.plans.view`)
  - Latihan Darurat (`emergency.drills.index`, permission `emergency.drills.view`)
  - Kontak Darurat (`emergency.contacts.index`, permission `emergency.contacts.view`)
- Menu ditempatkan di group `Operasional & Support`.
- Setiap menu memakai permission dedicated agar user yang hanya punya permission sebagian tetap melihat menu sesuai hak mereka.

### Track 2: Perbaiki Role Mapping Training

- Membuat bundle permission Training:
  - `$trainingFull`: 7 permissions (programs view/create/update, records view/create/update/export) — untuk manager/officer
  - `$trainingViewExport`: 3 permissions (programs view, records view/export) — untuk supervisor
- Menginjeksi bundle ke role map:
  - **QHSSE Manager**: menerima `$trainingFull` (7 permissions)
  - **QHSSE Officer**: menerima `$trainingFull` (7 permissions)
  - **Supervisor**: menerima `$trainingViewExport` (3 permissions)
- Tidak mengubah role lain (Employee, Contractor, Auditor, Top Management, dll) agar tidak mengganggu scope mereka.

### Track 3: Bungkus Emergency Routes dengan Auth Middleware

- Membungkus seluruh route Emergency Plan, Drill, dan Contact dengan middleware:
  - `auth` — memastikan user authenticated
  - `verified` — memastikan email verified
  - `active` — memastikan user active
- Konsisten dengan modul lain seperti Training, Asset, Contractor, dll.
- Controller policy tetap ditegakkan sebagai defense-in-depth.
- Anonymous user sekarang redirect ke login (HTTP 302), bukan 403.

### Track 4: Tambahkan Regression Test

- Membuat `tests/Feature/P01RegressionTest.php` dengan 6 test cases:
  1. `it redirects anonymous users to login for emergency routes` — memastikan anonymous redirect, bukan 403
  2. `it allows users with training permissions to access training routes` — memastikan user dengan permission dapat akses route
  3. `it allows users with emergency permissions to access emergency routes` — memastikan user dengan permission dapat akses route
  4. `it grants QHSSE Manager all training permissions` — memverifikasi role map dan user role assignment
  5. `it grants QHSSE Officer all training permissions` — memverifikasi role map dan user role assignment
  6. `it grants Supervisor training view and export permissions only` — memverifikasi supervisor tidak dapat create/update
- Test memakai format Pest functional, `beforeEach` seed `RolesAndPermissionsSeeder`.
- Total 71 assertions.

- Memperbarui `tests/Feature/NavigationConfigurationTest.php`:
  - Mengganti expectation dari 8 operational menu menjadi 12 operational menu.
  - Menambahkan assertion untuk 6 submenu baru dengan permission yang benar.

## 13. Decision Log

- **Pecah menu Training/Emergency menjadi submenu**: Memisahkan menu top-level menjadi item dedicated per submodul agar user dapat menemukan fitur dengan mudah. Training Records dan Emergency Drills yang sebelumnya hanya bisa dijangkau via URL atau link internal, sekarang punya menu langsung.
- **Training permission hanya untuk QHSSE roles**: Employee, Contractor, Auditor, Top Management tidak diberi training permission karena scope operasional mereka tidak mencakup pengelolaan training program. Jika diperlukan kelak, dapat disesuaikan via role matrix.
- **Supervisor training read-only**: Supervisor diberi view dan export, tetapi tidak create/update, karena peran mereka lebih monitoring daripada admin.
- **Emergency route dibungkus auth middleware**: Memilih konsistensi defense-in-depth dan UX redirect login daripada mengandalkan controller policy saja.
